# Content Research Tools / 内容素材工具

Use this reference when building content research, competitor analysis, script deconstruction, or素材库 workflows.

做内容研究、竞品拆解、脚本拆解或素材库流程时使用本文件。

## Core Rule / 核心原则

- Only download or store videos that the seller owns, has permission to use, or is saving for private research and note-taking.
- 只下载或保存卖家自己拥有、已获授权，或仅用于内部研究记录的视频。
- Do not present downloaded third-party videos as reusable commercial assets unless the user has rights to use them.
- 不要把下载的第三方视频写成可直接商用素材，除非用户拥有使用权。
- Do not encourage reposting other creators' videos, removing watermarks to hide origin, impersonation, or copyright/IP violations.
- 不鼓励搬运他人视频、去水印隐藏来源、冒充作者或侵犯版权/IP。

## Tool / 工具

GreenVideo: https://greenvideo.cc/

Recommended use:

- Save the seller's own TikTok, Instagram, YouTube Shorts, or other short-video assets into a local素材库.
- Download authorized creator videos for internal review, briefing, or ad/affiliate documentation.
- Save public competitor videos for private structure analysis: hook, opening shot, pain point, product demo, proof, offer, CTA, comments.
- Build a content research table with source URL, platform, creator, topic, hook, visual structure, claim risk, and “can use / cannot use” rights status.

建议用途：

- 保存自己账号的 TikTok、Instagram、YouTube Shorts 等短视频到本地素材库。
- 下载已授权达人视频，用于内部复盘、brief 或广告/联盟归档。
- 保存公开视频做内部拆解：开头钩子、首帧、痛点、产品演示、证明点、优惠、CTA、评论区反馈。
- 建立内容研究表：来源链接、平台、作者、主题、钩子、画面结构、宣称风险、能否使用的版权状态。

## Practical Workflow / 实操流程

1. Collect 20-50 public examples for one product angle.
2. Use GreenVideo only to save items needed for review or authorized use.
3. Rename files with date, platform, creator, product angle, and rights status.
4. Extract patterns into the research table instead of copying the video.
5. Create original scripts and footage plans based on the pattern, not the source file.

1. 一个产品角度先收集 20-50 条公开视频案例。
2. 只用 GreenVideo 保存需要复盘或已授权使用的内容。
3. 文件名写清日期、平台、作者、产品角度和权利状态。
4. 把规律拆到表格里，不要直接复制视频。
5. 根据规律重新写原创脚本和拍摄计划，而不是搬源文件。
